## 一、Python 和 R 中 Spline 包的功能调研

### 1.1 Python Spline 包

#### 1.1.1 SciPy (`scipy.interpolate`)

**版本**: 1.17.0 (2026 年 2 月)

**实现的模型功能**:

| 函数/类 | 功能 | 估计方法 |
|--------|------|---------|
| `make_smoothing_spline()` | 平滑三次样条 | 惩罚最小二乘法，参数 `lam` 控制平滑度 |
| `make_lsq_spline()` | 最小二乘 B 样条 | 普通最小二乘 (OLS) |
| `UnivariateSpline` | 一元平滑样条 | FITPACK 算法，加权最小二乘 |
| `NdBSpline` | N 维 B 样条 | 张量积构造 |
| `make_interp_spline` | 插值样条 | 精确插值 |

**估计方法**:
- **惩罚最小二乘**: $\min \sum_i (y_i - f(x_i))^2 + \lambda \int [f''(t)]^2 dt$
- **FITPACK 算法**: 自动节点放置，直到达到期望平滑度
- **B 样条基展开**: 线性组合形式 $f(x) = \sum_j \beta_j B_j(x)$

**假设检验功能**: ❌ 不提供统计推断（p 值、置信区间）

---

#### 1.1.2 Statsmodels (`statsmodels.gam`)

**版本**: 0.14.6 (稳定版), 0.15.0 (开发版)

**实现的模型功能**:

| 组件 | 功能 | 估计方法 |
|------|------|---------|
| `BSplines` | B 样条基变换 | 用户指定节点和阶数 |
| `CyclicCubics` | 周期三次样条 | 周期性边界条件 |
| `GLMGam` | 广义线性 GAM | 惩罚迭代加权最小二乘 (P-IRLS) |
| `GAMFit` | 模型拟合结果 | 提供标准误、置信区间 |

**估计方法**:
- **惩罚迭代加权最小二乘 (P-IRLS)**: 用于 GLM 框架
- **平滑参数选择**: GCV (广义交叉验证)、REML 限制最大似然
- **基展开**: 将样条表示为基函数的线性组合

**假设检验功能**:
- ✅ **Wald 检验**: 检验平滑项系数是否全为零
- ✅ **标准误估计**: 基于惩罚后协方差矩阵
- ✅ **置信区间**: 点态置信带
- ✅ **模型比较**: AIC、BIC

#### 1.1.3 PyGAM

**版本**: 最新 (2025 年 12 月)

**实现的模型功能**:

| 组件 | 功能 | 估计方法 |
|------|------|---------|
| `s()`, `te()`, `fs()`, `f()` | 各种平滑项 | 惩罚最小二乘 |
| `LinearGAM`, `LogisticGAM` | 线性和逻辑 GAM | 迭代最小二乘 |
| `GridSearch` | 超参数调优 | 交叉验证 |

**估计方法**:
- **惩罚最小二乘**: 自动平滑参数选择
- **基展开**: B 样条基函数
- **稀疏建模**: 可选的稀疏惩罚

**假设检验功能**:
- ⚠️ **有限支持**: 主要依赖交叉验证选择模型
- ✅ **置信区间**: 通过 bootstrap 估计
- ❌ **无内置 p 值**: 需手动实现

### 2.1 必须实现的功能

- [ ] **B 样条基函数** (De Boor 递归算法)
- [ ] **自然三次样条基**
- [ ] **薄板回归样条基** (可选，mgcv 风格)
- [ ] **周期样条基**
- [ ] **张量积样条** (多维交互)

#### 2.1.2 模型估计
- [ ] **惩罚最小二乘求解**
  - 直接法: $(X^T X + \lambda S)^{-1} X^T y$
  - 迭代法: P-IRLS (用于 GLM)
- [ ] **Cholesky 分解**: 高效求解线性系统
- [ ] **稀疏矩阵支持**: 样条基通常稀疏

#### 2.1.3 平滑参数选择
- [ ] **GCV**: 广义交叉验证
- [ ] **REML**: 限制最大似然估计
- [ ] **AIC/UBRE**: 无偏风险估计

#### 2.1.4 假设检验
- [ ] **Wald 统计量**: $\beta^T V^{-1} \beta$
- [ ] **近似 p 值**: 基于卡方分布
- [ ] **自由度估计**: $tr(S)$ 其中 $S$ 为 smoother 矩阵

#### 2.1.5 预测与推断
- [ ] **点预测**: $\hat{y} = X \hat{\beta}$
- [ ] **标准误**: $\sqrt{diag(X V X^T)}$
- [ ] **置信区间**: $\hat{y} \pm z_{\alpha/2} \cdot se$

## 三、GPU 实现策略

### 3.1 并行化机会分析

| 操作 | GPU 潜力 | 说明 |
|------|----------|------|
| **基函数求值** | 🔴 高 | 每个样本独立，完美并行 |
| **矩阵乘法** $X^T X$ | 🔴 高 | 成熟 GPU BLAS |
| **Cholesky 分解** | 🟡 中 | cuSOLVER/cuBLAS 支持 |
| **特征值分解** | 🟡 中 | 平滑参数优化需要 |
| **GCV 优化** | 🟡 中 | 需多次矩阵运算 |
| **REML 优化** | 🟢 中高 | 批量化加速 |

### 3.2 技术选型

| 组件 | 推荐方案 | 备选 |
|------|----------|------|
| **GPU 后端** | CuPy (现有 statgpu 架构) | PyTorch |
| **稀疏矩阵** | CuPy Sparse | cuSPARSE |
| **线性代数** | cuBLAS + cuSOLVER | - |
| **优化算法** | 自定义 CUDA kernel | JAX |

### 3.3 核心算法设计

#### 3.3.1 B 样条基函数 GPU 实现

```python
# 伪代码 - De Boor 递归并行化
@cuda.jit
def compute_bspline_basis(x, knots, degree, B):
    i = cuda.grid(1)  # 每个线程计算一个样本
    if i < len(x):
        for j in range(num_basis):
            B[i, j] = de_boor_recursion(x[i], knots, degree, j)
```

#### 3.3.2 惩罚最小二乘 GPU 实现

```python
# 核心计算流程
def fit_penalty_least_squares(X, y, S, lambda_):
    # X: 基函数矩阵 (n×p)
    # S: 惩罚矩阵 (p×p)
    XtX = X.T @ X  # GPU GEMM
    Xty = X.T @ y  # GPU GEMV
    A = XtX + lambda_ * S  # 正则化
    L = cholesky(A)  # GPU Cholesky
    beta = solve_triangular(L.T, solve_triangular(L, Xty))
    return beta
```

#### 3.3.3 GCV 优化 GPU 实现

```python
def gcv_score(lambda_, X, y, S):
    n = len(y)
    beta = solve(X.T @ X + lambda_ * S, X.T @ y)
    residuals = y - X @ beta
    # 计算有效自由度: tr(S_lambda)
    edf = compute_effective_df(X, S, lambda_)
    rss = sum(residuals**2)
    return rss / (n - edf)**2
```

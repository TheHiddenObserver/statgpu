- 信号强度：beta_scale=0.5 (弱信号 0-3 的β = 0.5, 1.0, 1.5, 2.0)

## 关键发现

### 1. Power (真正例检出率)

**两个后端都达到 100% power** - 在全部 50 次实验中检出所有 12 个真正信号。

### 2. 弱信号检测

两个后端在所有弱信号（特征 0-3）上都达到了 100% 检出率，即使是最弱的信号（特征 0, β=0.5）也能稳定检出。

### 3. FDR 控制

- NumPy: 平均 FDR = 7.2% ± 9.8%
- Torch: 平均 FDR = 6.4% ± 7.6%

两个后端的 FDR 都低于 10% 目标，控制良好。

### 4. 分离度

分离度 = 信号 |W| 均值 / 噪声 |W| 最大值

- NumPy: 79.4x ± 27.8x
- Torch: 74.8x ± 17.5x

分离度越高表示信号与噪声的可区分性越好。

### 5. 计算效率

- NumPy: 0.47 ± 0.46 秒/实验
- Torch: 0.46 ± 0.31 秒/实验

在此问题规模下（n=400, p=80），两个后端速度相当。

## 结论

1. **两个后端统计性能等效** - 在所有性能指标上均无显著差异

2. **100% Power** - 两个后端在所有 50 次实验中都检测出了全部 12 个真正信号

3. **弱信号检测优秀** - 即使是最弱的信号（特征 0, β=0.5）也能 100% 检出

4. **FDR 控制良好** - 平均 FDR (NumPy 7.2%, Torch 6.4%) 低于 10% 目标

5. **高分离度** - 信号 W 统计量是噪声最大 W 的约 75-80 倍

6. **运行时间相近** - 在此问题规模下，两个后端速度相当

## 建议

1. **CPU 场景使用 NumPy 后端** - 在中等规模问题上性能与 GPU 相当

2. **GPU 场景使用 Torch 后端** - 可能在更大规模问题（n>1000, p>200）上有优势

3. **两个后端都可用于生产环境** - 统计性能等效，FDR 控制良好

# statgpu Knockoff Filter: NumPy vs CuPy 50 实验对比报告

**日期:** 2026-04-19  
**实验次数:** 50 次独立实验  
**实验配置:**
- 数据规模：n=400, p=80
- 真正信号：特征 0-11 (12 个信号，强度递增)
- 目标 FDR: q=0.10
- 信号强度：beta_scale=0.5 (弱信号 0-3 的β = 0.5, 1.0, 1.5, 2.0)
- 噪声水平：noise_scale=0.5

## 对比的后端

1. **NumPy** - CPU 后端
2. **CuPy** - CUDA GPU 后端

*注：PyTorch GPU 后端因 device 一致性问题暂时未纳入对比，后续修复后会补充*

## 弱信号检测能力 (特征 0-3)

弱信号具有最小的系数 (β = 0.5, 1.0, 1.5, 2.0)，是最难检测的特征。

| 特征 | 信号强度 | NumPy 检出率 | CuPy 检出率 |
|------|----------|-------------|-------------|
| 0 | 0.5 | **100%** | **100%** |
| 1 | 1.0 | **100%** | **100%** |
| 2 | 1.5 | **100%** | **100%** |

**两个后端在所有弱信号上都达到了 100% 检出率。**

## 关键发现

### 1. Power (真正例检出率)

**两个后端都达到 100% power** - 在全部 50 次实验中检出所有 12 个真正信号。

### 2. 弱信号检测

两个后端在所有弱信号（特征 0-3）上都达到了 100% 检出率，即使是最弱的信号（特征 0, β=0.5）也能稳定检出。

### 3. FDR 控制

- NumPy: 平均 FDR = 6.7% ± 9.9%
- CuPy: 平均 FDR = 7.9% ± 9.9%

两个后端的 FDR 都低于 10% 目标，控制良好。

### 4. 分离度

分离度 = 信号 |W| 均值 / 噪声 |W| 最大值

- NumPy: 88.9x ± 33.1x
- CuPy: 89.7x ± 33.0x

分离度越高表示信号与噪声的可区分性越好。两个后端分离度非常接近。

### 5. 计算效率

- NumPy: 0.106 ± 0.007 秒/实验
- CuPy: 0.126 ± 0.130 秒/实验

在此问题规模下（n=400, p=80），两个后端速度相当。CuPy 的标准差较大是因为第一次运行时可能有 JIT 编译开销。

## 与之前 NumPy vs Torch 对比的一致性

**注意：** 时间差异主要是因为远程服务器使用编辑模式安装 statgpu，且可能使用了不同的优化配置。

## 结论

1. **两个后端统计性能等效** - 在所有性能指标上均无显著差异

2. **100% Power 达到** - 两个后端在所有 50 次实验中都检测出了全部 12 个真正信号

3. **弱信号检测优秀** - 即使是最弱的信号（特征 0, β=0.5）也能 100% 检出

4. **FDR 控制良好** - 平均 FDR (NumPy 6.7%, CuPy 7.9%) 低于 10% 目标

5. **高分离度** - 信号 W 统计量是噪声最大 W 的约 89 倍

6. **运行时间相近** - 在此问题规模下，两个后端速度相当

## 建议

1. **中等规模问题使用 NumPy 后端** - 在 n=400, p=80 规模下性能与 GPU 相当

2. **大规模问题考虑 CuPy 后端** - 当 n>1000, p>200 时 GPU 并行可能有优势

3. **两个后端都可用于生产环境** - 统计性能等效，FDR 控制良好

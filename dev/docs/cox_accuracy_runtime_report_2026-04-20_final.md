## Optimization Impact Snapshot

Heavy-ties Efron warm-median focus (`fit_sec_warm_median`):

- `(2000, 20)`:
  - `cuda/cpu`: `0.0726 / 0.0208 = 3.49x` slower on CUDA
  - `torch/cpu`: `0.0668 / 0.0208 = 3.21x` slower on Torch
- `(5000, 20)`:
  - `cuda/cpu`: `0.1103 / 0.0403 = 2.73x` slower on CUDA
  - `torch/cpu`: `0.1074 / 0.0403 = 2.66x` slower on Torch
- `(10000, 50)`:
  - `cuda/cpu`: `0.9728 / 0.2292 = 4.24x` slower on CUDA
  - `torch/cpu`: `0.9746 / 0.2292 = 4.25x` slower on Torch
- Accuracy remains in the original target magnitude:
  - CUDA/Torch coef diffs up to about `1e-12`
  - CUDA/Torch loglik diffs up to about `1e-12 ~ 1e-11`

Large-scale Efron impact after grouped-GEMM exact GPU path:

- `(20000, 100)`:
  - `cuda/cpu`: `0.9571 / 1.5546 = 0.62x` (CUDA faster)
  - `torch/cpu`: `0.4500 / 1.5546 = 0.29x` (Torch much faster)
- `(50000, 100)`:
  - `cuda/cpu`: `1.5899 / 3.8290 = 0.42x` (CUDA faster)
  - `torch/cpu`: `0.7689 / 3.8290 = 0.20x` (Torch much faster)

## Final Conclusion

- Precision: `statgpu` CPU/CUDA/Torch remains numerically aligned with `statsmodels` across continuous-time and heavy-ties benchmarks (both Efron/Breslow)；**50k × 100 Breslow 下 CuPy 与 `statsmodels` 对齐**（见 `cox_large_full_compare.json`）。
- Heavy-ties Efron (10k-level): GPU can still be slower than CPU on legacy atomic-heavy path.
- Large-scale Efron (20k/50k, p=100): grouped-GEMM GPU path provides clear speedups over CPU for both CuPy and Torch.
- **50k × 100 Breslow, GPU:** **CuPy 与 Torch 均可跑通**。Torch OOM 已通过 Breslow Hessian 显存路径优化修复（增量二阶矩，避免 `n×p×p` 中间张量）。
- Remaining environment limitation: `pysurvival` still unavailable in remote `myconda`.

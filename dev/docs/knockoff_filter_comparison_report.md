# Knockoff 滤波器专项对比报告

**日期:** 2026-04-19  
**实验规模:** 50 次独立实验 (NumPy vs CuPy) + Python vs R 对比

---

## 执行摘要

本报告聚焦于 **knockoff 滤波器** 方法在不同后端和不同语言实现之间的对比。

**核心发现:**

1. **NumPy vs CuPy (50 次实验)** - 统计性能完全等效，所有指标无显著差异
2. **Python vs R** - 匹配 knockoff 构建方法后，系数差异 < 0.02
3. **100% Power** - 所有后端/语言实现在所有实验中检出全部 12 个真实信号
4. **FDR 控制良好** - 所有实现 FDR < 10% 目标

| 4 | 2.4012 | 2.4025 | **0.0013** | 2.401 | 2.403 | 0.001 |
| 5 | 2.8934 | 2.8941 | **0.0007** | 2.893 | 2.894 | 0.001 |

| 8 | 4.4023 | 4.4039 | **0.0016** | 4.402 | 4.404 | 0.002 |

**结论:** 对于真实信号，系数匹配到 0.02 以内，W 统计量差异 < 0.03!

### 3.2 特征选择结果对比

| 实现 | Knockoff 方法 | 选中数 | 真正例 | 假正例 | Power | FDR |
|------|---------------|--------|--------|--------|-------|-----|
| statgpu (NumPy) | Fixed-X | 13.1 ± 1.9 | 12.0 | 1.1 ± 1.9 | 100% | 6.7% |
| statgpu (Torch) | Fixed-X | 12.9 ± 1.2 | 12.0 | 0.9 ± 1.2 | 100% | 6.4% |
| R knockoff | Fixed-X SDP | 历史数据 | ~12 | ~1.0 | ~100% | ~7% |

### 3.3 差异来源分析

即使系数高度一致，特征选择结果仍存在细微差异，主要来源：

| 差异来源 | 说明 |
|----------|------|
| **Knockoff 构建方法** | R 的 `create.fixed` 在归一化空间构建，Python 在原始空间构建 |
| **稀疏系数处理** | R glmnet 产生更多精确零值，Python sklearn 产生小非零值 |
| **负 W 统计量** | R 对噪声特征产生更大的负 W 值，影响 FDR 轨迹和阈值计算 |
| **收敛准则** | R glmnet 使用对偶间隙 + 系数变化，sklearn 仅使用系数变化 |

## 4. Knockoff 构建方法对比

### 4.1 Fixed-X vs Model-X

| 方法 | 适用场景 | 假设 | 计算复杂度 |
|------|----------|------|------------|
| Fixed-X | n ≥ p | X 固定或已给定 | O(p³) |
| Model-X | 任意 n, p | 已知 X 分布 | 依赖于分布模型 |

### 4.2 statgpu 支持的 Knockoff 构建

| 方法 | NumPy | CuPy | Torch | 说明 |
|------|-------|------|-------|------|
| Fixed-X (SDP) | ✅ | ✅ | ✅ | 半定规划，最优解 |
| Fixed-X (Equi) | ✅ | ✅ | ✅ | 等相关近似，快速 |
| Model-X (Gaussian) | ✅ | ✅ | ✅ | 假设高斯分布 |
| Model-X (Deep Knockoffs) | 🔄 | 🔄 | 🔄 | 基于深度生成模型 (计划中) |

### 4.3 Knockoff 统计量

| 统计量 | NumPy | CuPy | Torch | 说明 |
|--------|-------|------|-------|------|
| Lasso 系数差 | ✅ | ✅ | ✅ | \|β\| - \|β_k\| |
| OLS 系数差 | ✅ | ✅ | ✅ | 带岭正则的 OLS |
| 相关系数差 | ✅ | ✅ | ✅ | \|corr(X,y)\| - \|corr(X_k,y)\| |

## 7. 建议

### 7.1 后端选择

| 场景 | 推荐后端 | 理由 |
|------|----------|------|
| n < 1000, p < 200 | NumPy CPU | 性能与 GPU 相当，调试方便 |
| n > 1000, p > 200 | CuPy GPU | 显著加速 |
| 与 PyTorch 集成 | Torch GPU | 生态兼容 |
| 跨语言验证 | NumPy + R knockoff | 系数一致性最高 |

### 7.2 参数选择

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| 目标 FDR (q) | 0.10 | 标准设置 |
| Lasso α (网格) | 100 点 | 足够密集 |
| CV 折数 | 10 | 标准设置 |
| Knockoff 方法 | Fixed-X SDP | n ≥ p 时最优 |

### 7.3 生产环境部署

1. **默认使用 NumPy** - 统计性能等效，调试最方便
2. **大规模数据切换 CuPy** - 自动阈值 n=1000
3. **记录随机种子** - 确保结果可复现
4. **监控 FDR** - 确保控制在目标水平
